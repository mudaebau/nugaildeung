## 차수 1~10 (v2.3) — 재확인

이전 점검(design-audit.md 1차)에서 확인된 내용은 이번 갱신 시점 코드에서도 회귀 없이 유지됨을 재확인했다. 상세 내용은 변경 없음 — 히어로 배지 로고(차수1), 서브탭 A안 기본 구조(차수2, 이번 차수18에서 헤더 배너 스타일이 추가로 얹혔을 뿐 탭 자체 구조는 불변), 단계 중심 관리홈(차수3), 스코어·리더보드 단계 구분(차수4), 참가자 탭 체크인·입금(차수5, 이번 차수19에서 참가자 추가 방식만 개편), 대회 정보 수정 3등급(차수6), 위저드 3건(차수7), 전광판 B안(차수8), 기록 입력 방식 옵션(차수9), 전광판 범위 축(차수10) 모두 유지.

## 차수 17 — 긴급 버그 5건

- **[S10] 본선 선수별 스코어 입력 불가**: `renderScoreTab()`의 `scoreStageId`가 완료된 단계에 고착되던 문제 원인 규명 및 수정. `mgShow('mgScore')` 진입 시 `scoreStageId=null` 리셋으로 항상 활성 단계 기본 표시. 리더보드 탭의 `boardStageSel`도 동일 패턴이라 함께 수정.
- **[S9][S7] 조/코스 선택 칩 잘림·터치 문제**: `.chiprow`에 `touch-action:pan-x` 추가.
- **[S5] 조편성 자동 실행**: 참가자 추가/삭제/승격 4곳의 자동 `makeGroups()` 호출 제거, 버튼(`조편성하기`)으로만 실행되도록 복원.
- **[S1] 히어로 배지 로고 컬러 미표시**: SVG `fill` 자체는 원래부터 정상이었음을 확인. 실제 결함은 Google Fonts 요청에 `Noto Sans KR` weight 900이 누락되어 있던 것 — 요청 URL에 `;900` 추가, `document.fonts`로 실제 로드 확인.

## 차수 18 — 서브탭 A안 정합 + 헤더 URL 노출 제거

**사전 비교 결과(목업 대비 차이점)**:
- 서브탭 자체(아이콘 7개·순서·언더라인·배지)는 차수2에서 이미 목업과 정확히 일치 — 실질 차이는 라벨 `font-size`(11px→11.5px 목업과 통일)뿐.
- 헤더(`​.mgmthead`)는 목업이 딥블루 배너+흰 글자인 반면 기존 구현은 흰 배경+어두운 글자였음 — 운영자 확인 후 목업과 동일하게 딥블루 배너로 변경.

**적용 내용**:
- `.mgmthead`를 딥블루 배너(`background:var(--main-deep)`, 흰 텍스트)로 변경, 탭 바로 위까지 풀블리드.
- `.tabA .ti span` font-size 11px→11.5px로 목업과 일치.
- 헤더 URL 노출 전수 제거: 전광판(`#boardHeadName`, 기존 `nuga1.kr/t/1024/board` 하드코딩 제거) · 참가 신청(`#applyUrlHint`, 기존 `nuga1.kr/t/${id}/apply` 동적 노출 제거) · 심판(`#judgeHeadName`, 기존 "실서비스: 심판은…" 안내문 제거) 3곳 모두 대회명으로 대체. 링크 공유는 기존처럼 관리홈 도구 행·각 탭 [공유] 버튼으로만 가능(변경 없음).

## 차수 19 — 참가자 구조 개편

- **개인/단체 추가 통합**: `[참가자 추가]` 모달에 `[개인 추가|단체 추가]` 세그먼트 추가. 단체 추가는 기존 붙여넣기 파서(`parsePaste`/`confirmPaste`) 그대로 재사용, 완료 후 모달이 닫히고 명단으로 복귀. 상시 노출되던 붙여넣기 카드는 명단 탭에서 제거.
- **단계별 명단**: 참가자 탭 기본 명단 = 현재 활성 단계(`playersActiveStage`), 이전 단계는 단계 칩(`stageChipsHTML` 재사용)으로 조회 전용 열람. 콜업(`set_player_stage`)·제외(기권 처리 + 사유를 `tournament_edit_logs`에 기록) 가능. `[명단 확정]`(`confirm_stage_roster`) 전에는 조편성 진입 시 안내 카드로 차단.
- **S10 재발 방지 규칙 적용**: 참가자 탭의 `playersStageId`도 `mgShow('mgPlayers')` 진입 시 리셋되어 항상 활성 단계 기본 표시 — 스코어/리더보드 탭과 동일한 원칙.

## 차수 20 — 선수별 입력 코스 구조

- 코스 2개 이상: 선수별 그리드에 코스 칩(`.chiprow` 재사용, 차수17 `touch-action:pan-x` 유지) 선택 후 해당 코스만 표시.
- 18홀 코스는 전반 9홀/후반 9홀 두 섹션 + 각 소계 + 코스 합계. 코스 길이가 18이 아닌 경우(방어적 처리) 단일 섹션.
- 그리드 상단에 코스별 소계 나열 + 총합 고정 표시(`.gridsummary`, 새 클래스 — kv행/표 형태 아닌 인라인 요약바).
- 1코스 대회는 코스 칩·요약 생략, 기존처럼 단일 그리드만(전반/후반 분리는 유지, 회귀 아님).

## 차수 22 — 브랜드 v3(말풍선 로고) 적용

- **적용**: 앱바 = `logo-appbar.svg`(무풍선 변형), 홈 히어로·요강·신청 페이지 = `logo-main.svg`, 전광판 푸터·앱바 아이콘(`.logo .flag`) = `app-icon.svg`(1-풍선), 파비콘·앱 아이콘은 동일 파일명으로 내용만 교체되어 코드 변경 불필요.
- **`--navy` 토큰화**: `:root`에 `--navy:#1D3D5C` 등록, 하드코딩돼 있던 `.livecard` 그라디언트의 `#1D3D5C`를 `var(--navy)`로 치환(색상 값 자체는 불변). `--board-bg`(#173350)는 별개 기존 토큰이라 손대지 않음.
- **구 로고 참조 전수 검색**: `logo-badge`/`logo-symbol`/`logo-horizontal`/`#224670`(구 배지 배경색) 검색 결과 **잔존 0건**. `brand/` 참조 5종(`app-icon.svg`/`apple-touch-icon-180.png`/`favicon-32.png`/`logo-appbar.svg`/`logo-main.svg`) 전부 실제 파일 존재 확인, 브라우저에서 `naturalWidth/naturalHeight`로 4곳(앱바·홈 히어로·요강·신청) 전부 정상 로드 확인.
- 홈 히어로의 구 v2 배지 인라인 SVG(수기 제작 사각형 조합 + `font-weight="900"` 텍스트)는 완전히 제거되고 `<img src="brand/logo-main.svg">`로 대체 — 이 900 굵기가 코드에 남아있던 유일한 지점이었고, 이번에 제거되며 900 예외 자체가 소멸했다.

## 규칙 위반 검사 (재스캔)

### 표 폐지 — 완료 (잔여 0건)

`<table` 검색 결과 인쇄용 결과지(`#rtablePrint`, `display:none`+`@media print`) 1건만 존재, 변경 없음. 차수17~20에서 추가된 UI(`.gridsummary`, 명단 확정 카드, 콜업/제외 오버레이 등)는 전부 `.card`/`.toolrow`/인라인 `<span>` 조합으로 구성되어 표·kv행 신규 없음.

### 옐로우 사용 — 완료 (위반 0건)

`--sun`/`#F7CF5C` 검색 결과 전부 1등(`top1`/`lead`/`pod.first`) 또는 "내 것"(`minecard`/`mycard` 배지·테두리) 맥락, 로고 심벌(브랜드 자산 예외) 한정. `.btn:focus-visible` 아웃라인은 키보드 접근성 포커스 링 용도로 장식적 강조가 아니라 규칙 적용 대상 밖으로 판단.

### 폰트 굵기 — 완료 (위반 0건, 900 예외 폐기)

`font-weight:800|900` 전수 검색 결과 **0건**. 차수17까지 유일하게 남아있던 예외(히어로 배지 로고 SVG 워드마크 `font-weight="900"`)가 차수22에서 해당 인라인 SVG 자체가 `brand/logo-main.svg`(워드마크 700 고정, `brand/README.md` 규격)로 교체되며 제거됐다 — 이제 UI 텍스트와 로고 워드마크 모두 예외 없이 최대 700이다.

### 전광판 컬럼 폭 — 완료 (고정 px 금지 확인)

`BOARD_COLS`/`boardCols()` 모두 fr 비율 기반, 변경 없음.

### tournaments UPDATE RLS 금지 — 완료 (위반 0건)

`supabase/migrations/*.sql` 전체(0001~0033) 재검색 결과 `tournaments` 테이블에 대한 anon UPDATE 정책 0건. 차수16(보안)에서 도입한 `assert_owner` 기반 RPC 체계와 차수21에서 확장한 players/stages RPC 전환 이후에도 이 원칙은 그대로 유지된다.

## 결론

v2.3 차수 1~10, 피드백 1차 차수 17~20, 차수22(브랜드 v3)의 확인 조건을 모두 충족했다. 서브탭 자체는 이미 목업과 일치했고 헤더 배너·URL 노출도 이전 갱신에서 정리됐다. 이번 갱신에서는 브랜드 v3 리브랜딩으로 구 로고 자산 참조가 전부 새 자산으로 교체됐음을 확인했고, 그 과정에서 유일하게 남아있던 900 굵기 예외(히어로 로고 워드마크)가 자연스럽게 소멸해 폰트 굵기 규칙이 이제 예외 없이 전면 적용된다. 공통 규칙 5건 전수 재스캔 결과 잔여 위반 0건.

남은 항목: 점검표 차수 14~16(개인정보·오프라인·보안★, 이미 별도로 완료·검증됨), 차수21(보안 RPC 전환, 완료), 리허설 및 파일럿.
