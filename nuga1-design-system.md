# 누가일등 디자인 시스템 v2

> 이 문서는 index.html에 실제로 구현되어 있는 디자인 규칙을 코드에서 그대로 추출해 기록한 참조 문서다.
> 새 화면·컴포넌트를 만들 때는 새 스타일을 발명하지 말고 이 문서의 변수·클래스를 재사용한다.

## 1. 색상 (index.html `:root` CSS 변수)

| 변수 | 값 | 용도 |
|---|---|---|
| `--main` | #4E8FD0 | 메인 블루 — 버튼, 앱바 배경, 강조 |
| `--main-deep` | #2F6BAA | 진한 블루 — 헤더 텍스트, 배지, 강조 텍스트 |
| `--main-soft` | #E3EEF9 | 연한 블루 — 선택된 배경, ghost 버튼 배경 |
| `--bg` | #F2F5F9 | 기본 화면 배경 |
| `--paper` | #FFFFFF | 카드·입력창 배경 |
| `--ink` | #2B3A4A | 본문 텍스트 |
| `--muted` | #68798C | 보조 텍스트 |
| `--line` | #DCE6F0 | 테두리 |
| `--mint` / `--leaf` | #57C2A8 / #7FBF6A | 여유 색상 (현재 제한적으로 사용) |
| `--sun` | #F7CF5C | 옐로우 — **"1등/내 것" 단일 강조 전용** |
| `--coral` | #EF8368 | LIVE 배지, 경고성 강조 |
| `--red` | #E2604A | 삭제·위험 액션 |
| `--board-bg` | #173350 | 전광판 전용 딥 네이비 배경 |
| `--board-line` | #2C5680 | 전광판 전용 구분선 |

**옐로우 사용 규칙**: 1위 표시, "내가 개설한 대회" 배지 등 화면당 하나의 강조에만 쓴다. 두 번째 강조색이 필요해도 옐로우를 늘리지 말고 `--coral` 등 다른 색을 쓴다.

## 2. 타이포그래피

| 서체 | 굵기 | 용도 |
|---|---|---|
| Noto Sans KR | 400/500/600/700 | 본문·UI 전체 (기본 서체) |
| Noto Sans KR 900 | 900 | **로고 워드마크 전용.** 본문·버튼·헤더에는 쓰지 않는다 |
| Oswald | 500/600/700 | 숫자·통계 (`.num` 클래스) — 접수번호, 스코어, 시계, 파(par) 등. `letter-spacing:.02em`, tabular-nums |
| Black Han Sans | — | 전광판(`.boardtop h3`, `.brow .name`) 전용 대형 타이틀·선수명. 일반 화면에는 쓰지 않는다 |

- 기준 크기: `html{font-size:20px}` (rem 스케일의 기준).
- 섹션 제목(`h2.sec`): 26px / 700 / `letter-spacing:-.02em`.
- **본문·버튼·헤더에 800 이상 굵기를 쓰지 않는다** (900은 로고 전용).

## 3. 레이아웃

- 모바일 프레임 `.mobile`: `max-width:430px`, 중앙 정렬, 배경 `--bg`.
- 전광판만 예외: `.mobile.wide` — 풀와이드, 배경 `--board-bg` (딥 네이비).
- 카드 `.card`: 흰 배경, 1px `--line` 테두리, `--radius`(20px) 라운드, 20px 패딩, 아래 여백 14px.
- 히어로 카드 `.poster`: 블루 그라디언트(`--main` → `--main-deep`), 흰 텍스트, 대회명·일정처럼 화면 최상단에 강조할 정보 1건에 사용 (예: 참가 신청 페이지, 대회요강 페이지).

## 4. 컴포넌트

| 클래스 | 규칙 |
|---|---|
| `.btn` | 최소 높이 56px(큰 터치 타깃), 16px 라운드, 700 굵기, 기본 배경 `--main` |
| `.btn.ghost` | 배경 `--main-soft`, 텍스트 `--main-deep` — 보조 액션 |
| `.btn.warn` | 배경 `--coral` — 삭제 등 위험 액션 |
| `.btn.full` / `.btn.big` / `.btn.sm` | 너비 100% / 높이 64px / 높이 46px(보조 액션) |
| `.pill` | 완전 라운드(99px) 배지. 변형: `.amber`(대기/주의), `.red`(경고), `.gray`(비활성) |
| `.chip` / `.fchip` | 완전 라운드 선택형 버튼(필터, 홀/코스 선택 등). 선택 시 `.on` |
| `.fl` | 카드 내부 입력 필드/정보 항목 라벨 (작은 라벨 + 본문) |

**터치 타깃**: 시니어 사용자를 고려해 보조 버튼도 최소 46px(`.sm`) 이상을 유지한다.

## 5. 브랜드 로고

- 원본 자산: `brand/` 폴더 (`logo-symbol.svg`, `logo-horizontal.svg`, `logo-horizontal-white.svg`, `symbol-white.svg`, `favicon-32.png`, `apple-touch-icon-180.png`, `app-icon.svg`/`app-icon-512.png`).
- 어두운 배경(앱바, 전광판): `symbol-white.svg` 또는 `logo-horizontal-white.svg`.
- 밝은 배경: `logo-symbol.svg` 또는 `logo-horizontal.svg`.
- 심벌 최소 표시 크기 16px. 그 이하에서는 favicon 단순화 버전을 쓴다.

## 6. 새 화면을 만들 때 원칙

1. 새로운 표(table)나 key-value 행 패턴을 새로 만들지 않는다 — `.card` + `.fl` 라벨 조합으로 표현될 수 있는지 먼저 검토한다 (예: 참가 신청·대회요강 페이지는 모두 이 조합만으로 구성됨).
2. 화면 최상단에 강조할 정보(대회명·일정 등)가 있으면 `.poster`를 재사용한다.
3. 공개 링크 페이지(`?apply=`, `?board=`, `?j=`, `?notice=`)는 로그인 없이 접근 가능해야 하며, 하단 내비게이션 바 없이 단독 화면(`.mobile` 최상위 sibling)으로 구성한다.
4. 옐로우 남용 금지 · 800 이상 굵기 금지 · 사진(스코어카드 등) 무검수 저장 금지 — 이 세 가지는 이후 어떤 Phase에서도 예외 없이 지킨다.
5. **단계(stage)를 기억하는 모든 UI 상태**(예: `scoreStageId`, `boardStageSel`)는 "저장값이 있으면 유지"가 아니라 **"화면 진입 시 현재 열린 단계와 다르면 활성 단계로 리셋"**이 기본이다 — 완료된 단계는 삭제되지 않고 `status`만 바뀌므로, 저장값 유지 방식은 새 단계가 열려도 과거 완료 단계에 고착되는 버그를 만든다(차수17 [S10] 원인). 단계 칩을 눌러 과거 단계를 조회하는 선택은 그 화면에 머무는 동안만 유지하고, 탭을 벗어났다 재진입하면 다시 활성 단계로 리셋한다.
